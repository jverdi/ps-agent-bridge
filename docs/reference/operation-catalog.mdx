For argument-level docs and copy/paste payloads for every operation, see [Operation arguments and examples](/reference/operation-arguments-and-examples).

`setTextStyle` supports `text`, `fontSize`, `fontName`/`font`, text color (`textColor`/`color`), text alignment (`alignment`/`align`/`justification`), `position`, bounds fitting (`maxWidth`, `maxHeight`, `minFontSize`, `hardMinFontSize`), overflow handling (`overflow: "resize" | "clip"`, `ellipsis`), and overlap avoidance (`avoidOverlapWith`, `overlapGap`).

`createShapeLayer` supports rounded corners (`cornerRadius`) on rectangle shapes and gradient fill via `fillType: "gradient"` + `gradient` config.

## Known behavior notes

- BatchPlay errors are inspected for embedded descriptor errors and surfaced with indexed failure detail.
- BatchPlay `textStyleRange[*].textStyle` replaces style objects; include full style fields when recoloring/re-styling text.
- Layer-targeted export may use Photoshop-controlled filenames depending on command path.
- `exportArtboards` defaults to a fallback document export when no descriptor/commands are provided.
- Document save without output can skip to avoid Save As dialogs when no existing path is present.
- `createTextLayer.position.y` is the text baseline, not glyph-top.
- Keep dependent work in one transaction (`psagent op apply`) because active document focus can shift between separate CLI invocations.
